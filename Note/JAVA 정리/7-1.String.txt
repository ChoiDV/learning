<API - "문자열">
문자열에 관련된 대표적 클래스는 String 클래스이다.

'String은 객체 자료형이다.'
객체 데이터는 생성자(new)를 사용해서 초기화를 해야하지만
자바에서 String은 기초데이터를 만들 때 처럼 String str = "Hello"; 처럼 초기화 할수 있게 했다.  

하지만 String str = "Hello"; 와 String str = new String("Hello"); 는 차이가 있다.
new로 생성하지 않고 ""를 사용한 문자열 상수는 자동으로 힙영역에 String 객체가 생성된다. 이미 존재하는 문자열 상수를 다시 사용한다면
이미 생성된 문자열 상수를 공유한다.  

<String의 기능>
String에는 많은 기능들(메소드) 들이 있다. (source에서 다루기)
String클래스의 치명적인 단점은 메모리를 과소비 하는 것이다. 
String 객체의 경우 처음 초기화된 데이터에 변화가 생기면 기존 것을 재활용 하지 않고 
기존의 것은 버리고 새로운 메모리를 이용한다. 
그래서 문자열 조작을 많이 하는 프로그램에서 이런 메소드를 많이 사용하면 비효율적이다. 

그래서 나온 클래스들이

<StringBuffer와 StringBuilder> 이다. 

이 클래스들은 객체 내부에 있는 버퍼에 문자열의 내용을 저장해 두고
 그 안에서 추가, 수정, 삭제 작업을 한다. 
그렇기 때문에 새로운 객체를 만들지 않고도 문자열 조작을 할 수 있다.
