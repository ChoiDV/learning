<form>
입력양식태그 (form) 태그: 입력 양식은 사용자에게 입력 받는 공간을 의미한다.
form 태그는 입력 양식을 만들 때 사용하는 태그이다.
<form 속성1=속성값 속성2=속성값...>
        <input type=" ? " name="전달할 파라미터 이름" ...>
</form> 의 방식으로 꼭 form 태그 안에 input 태그를 넣어서 사용한다.

form태그의 속성
1. action : 입력 데이터의 전달 위치를 지정
        -action=" ? "  : ?에 전달 위치를 적어준다.	
2. method: 입력 데이터의 전달 방식을 선택
        -get: url에 포함되어 전송(query String 형식)
        -post : 입력한 데이터가 눈에 보이지 않게 전송
		(url이 아니라 데이터 영역을 이용해서 데이터를 전송한다.)
<input>
form 태그안에 데이터를 입력할 수 있는 공간을 만든다.

<input type=" ? " name="전달할 파라미터 이름" ...> 으로 사용한다


input태그의 type들
1.button : 버튼

2. radio : 중복선택이 불가한 체크박스 (ex 남, 여 둘중 하나를 체크)
	type이 checkbox인것 하나당 체크박스가 하나가 생성되고 선택시 
	그 checkbox의 value값이 전달된다.

3.file: 파일 입력 양식을 생성

4.image: 이미지 형태를 생성

5. password : 비밀번호 입력 양식을 생성
	-비밀번호를 입력하는 창을 만들때는 이 타입을 선택하면 겉으로는 *으로 보인다.

6. text : 텍스트를 받을수 있다.

7. tel : 전화번호 양식을 생성해준다.  ( 공간-공간-공간 )

8. number : 숫자만 입력 할수있는 공간을 생성해주고 올리거나 내릴수 있는 버튼이 같이 생긴다.

9.email :  abc123@abc123 의 형식으로 입력해야하는 공간이 생성된다.

10. checkbox : 중복 선택이 가능한 체크박스 name은 같게하고 value는 각각의 value를 적어준다.

11. reset : 초기화 버튼

12. submit : 제출 버튼. 이버튼을 누르면 form태그에서 지정해준대로 데이터를 전달한다.

입력태그의 type들
1. select : 목록을 정해놓고 선택하는 양식 (ex 회원가입시 메일 뒷부분 선택하는창 )
    -  option 으로 목록을 추가할 수 있다.

2. textarea : 텍스트를 입력할수 있는 공간을 크게 만들어준다.

<input 태그의 속성>
1. autofocus : 포커스 자동 설정

2. placeholder : 각 필드에 대한 입력 형식 힌트제공
	ex: 입력창 위에 텍스트를 넣어서 힌트를 제공하는것, 사용자가 입력시 이 글은 사라진다.

3. required : 필수 입력 필드
 	(ex 회원가입창에서 필수 입력사항들 (아이디, 비밀번호, 이름 등등 입력하지않으면 넘어갈수 없다)

4. min , max : 숫자의 범위를 설정

5. pattern : 조건을 사용한 일반식 표시
	정규표현식을 사용해서 데이터의 유효성을 체크한다.
	ex) 전화번호 :  pattern="\d{2,3}[ .-]{0,1}\d{3,4}[ .-]{0,1}[0-9]{4}"
	      이메일   :  pattern="[a-zA-Z0-9][a-zA-Z0-9_.]+@[a-zA-Z]\w+(\.[a-z]+)+"
	을 사용해서 입력값의 유효성을 체크한다.
